# دليل الهوية والواجهات — منصة مقارنة أسعار الجوالات والحجز المسبق

> ملف جاهز للاستخدام مع Claude Design. الطريقة: الصق قسم **الهوية البصرية** كاملًا مرة واحدة في بداية الجلسة، بعدين الصق وصف صفحة واحدة فقط في كل رسالة. لا تعطيه صفحتين مع بعض — النتيجة بتصير باهتة.

---

## 1. الهوية البصرية

### الفكرة
العلم السوري مبني على **تراتب أفقي**: أخضر فوق، أبيض بالوسط، أسود تحت، وثلاث نجمات حمراء بقلب الأبيض. المنصة بتقرأ نفس التراتب حرفيًا من فوق لتحت: **رأس أخضر → جسم أبيض → تذييل أسود**، والنجوم الحمراء الثلاث محجوزة كعنصر توقيعي واحد ما بينستهلك بالزينة.

### التوكنز اللونية

| التوكن | Hex | الاستخدام |
|---|---|---|
| `--green` | `#007A3D` | الأزرار الأساسية، الرأس، انخفاض السعر، "أقل سعر" |
| `--green-deep` | `#00532A` | Hover، التذييل الثانوي، العناوين على أخضر |
| `--green-tint` | `#E6F2EB` | خلفية شارات التوفير، تمييز أفضل قيمة بالمقارنة |
| `--ink` | `#0E1311` | النص الأساسي، التذييل، الأسطح الداكنة |
| `--paper` | `#FFFFFF` | خلفية المحتوى |
| `--surface` | `#F4F6F5` | خلفية الصفحة، الجداول المخططة |
| `--line` | `#DDE3E0` | الحدود والفواصل |
| `--muted` | `#6B7772` | نص ثانوي، تواريخ، طوابع زمنية |
| `--star` | `#CE1126` | الأحمر النجمي — استخدام مقيّد (تحت) |
| `--star-tint` | `#FDECEE` | خلفية تحذير / عدّاد |

**قاعدة الأحمر (مهمة):** الأحمر لا يُستخدم إلا لما يكون في **خسارة أو وقت بينفد**:
1. العدّاد التنازلي لصلاحية الحجز.
2. ارتفاع السعر.
3. الحذف والأخطاء المدمّرة.
4. "آخر قطعة" / نفاد الكمية.

ممنوع أحمر بالأزرار العادية، ممنوع أحمر للإشعارات المحايدة. النسبة المستهدفة: **60% أبيض/رمادي، 30% أسود/رمادي داكن، 8% أخضر، 2% أحمر**.

### الخطوط

| الدور | الخط | الأوزان |
|---|---|---|
| العناوين | `Tajawal` | 700 / 800، تتبّع حرفي مضغوط `-0.02em` |
| النص | `IBM Plex Sans Arabic` | 400 / 500 |
| **الأسعار والأرقام** | `IBM Plex Mono` | 500 |

الـ Mono للأسعار مش زينة: بيخلّي خانات الأرقام تنحاذى عموديًا بجدول المقارنة والمتاجر، فالعين بتقارن `1,299` مع `1,150` بدون ما تعدّ الخانات.

**السلّم:** 40 / 32 / 24 / 18 / 16 / 14 / 12 px. سطر النص 1.7 (العربي بحاجة مساحة أكتر).

### الشبكة والشكل
- RTL بالكامل. الأيقونات الاتجاهية معكوسة. الأرقام والأكواد LTR داخل حاويتها.
- وحدة المسافة 4px. عمود المحتوى 1200px بحد أقصى.
- الحواف: بطاقات 12px، حقول 8px، شارات 999px.
- الظلال شبه معدومة: `0 1px 2px rgba(14,19,17,.06)`. الاعتماد على الحدود مش الظلال.

### العنصر التوقيعي: النجوم الثلاث
النجمة الخماسية الحمراء هي علامة المنصة، ولها **استخدامان اثنان فقط**:
1. **مؤشر خطوات الحجز** — ثلاث نجمات مفرغة تتحول لمصمتة حمراء مع كل خطوة (اختيار → تأكيد → استلام).
2. **شارة "تقييم موثّق"** — نجمة حمراء صغيرة مصمتة بجانب اسم المُقيّم.

**قرار مقصود:** التقييمات **لا تُعرض بنجوم**. بتنعرض كرقم (`4.6`) + شريط أفقي + عدد المراجعات. سبب: النجمة محجوزة للهوية، وما بنسمح بتخفيف قيمتها بتكرارها 500 مرة بالصفحة.

## 2. الصفحات

### 2.1 الصفحة الرئيسية — Homepage

**الهدف:** يفهم الزائر خلال 3 ثوانٍ إن هاي منصة أسعار **حيّة**، مش كتالوج.

**البطل (Hero):** مش صورة جوال وعنوان تسويقي. البطل هو **شريط الأسعار الحيّ**:
سطر واحد أفقي متحرك ببطء، عليه 8–10 منتجات مع سعرها بخط Mono وسهم أخضر للنزول / أحمر للصعود ونسبة التغير. تحته مباشرة حقل بحث ضخم (56px ارتفاع) بحدود خضراء 2px. تحته نص صغير رمادي: «آخر تحديث للأسعار قبل ٧ دقائق» مع نقطة خضراء نابضة.

```
┌──────────────────────────────────────────┐
│  [رأس أخضر: شعار | تصفح | البائعون | دخول]│
├──────────────────────────────────────────┤
│  ‹ iPhone 15  1,299 ▼2.4%  ·  S24  ... › │  ← تيكر
│                                          │
│        ابحث عن جوال، أو الصق رابط        │
│   ┌────────────────────────────────┐ [Q] │
│   └────────────────────────────────┘     │
│        ● آخر تحديث قبل ٧ دقائق           │
├──────────────────────────────────────────┤
│  أكبر انخفاض اليوم   [بطاقة][بطاقة][بطاقة]│
│  تصفّح حسب الفئة     [شرائح دائرية]      │
│  الأكثر مقارنة       [شبكة 4 أعمدة]      │
├──────────────────────────────────────────┤
│  [تذييل أسود: عن المنصة | للبائعين | ...] │
└──────────────────────────────────────────┘
```

**تفاصيل:**
- قسم «أكبر انخفاض اليوم» بخلفية `--green-tint` — القسم الوحيد الملوّن بالصفحة.
- شريط ثقة رفيع فوق التذييل: عدد المتاجر · عدد المنتجات المتتبَّعة · عدد نقاط السعر المسجّلة. أرقام Mono.
- **الحالة الفارغة للتيكر** (لو التحديث فشل): «الأسعار قيد التحديث. آخر نجاح: أمس ١١:٤٠» — رمادي، بدون أحمر. فشل داخلي مش خطر على المستخدم.

### 2.2 البحث والنتائج — Search & Filters

**الهدف:** الوصول لقائمة مرشّحة بأقل عدد نقرات، مع اقتراحات فورية.

**اقتراحات البحث الحيّة (Live Suggestions):** قائمة منسدلة مقسّمة بعناوين صغيرة: `منتجات` / `علامات` / `متاجر`. كل صف: صورة مصغّرة 32px + الاسم + أقل سعر بالـ Mono على اليسار. الحرف المطابق مظلّل بـ `--green-tint`.

**الفلاتر (عمود يمين، 280px، sticky):**
- نطاق السعر (شريط مزدوج، المقبض أخضر)
- العلامة التجارية (بحث داخلي + checkbox)
- الذاكرة / التخزين / حجم الشاشة / البطارية (شرائح قابلة للاختيار المتعدد)
- المتجر
- ☑ متاح للحجز المسبق فقط
- ☑ انخفض سعره خلال ٧ أيام

**بطاقة المنتج:**
```
┌─────────────────────┐
│      [صورة]      ♡ │
│ Galaxy S24 Ultra    │
│ ابتداءً من          │
│ 1,150 ر.س   ▼ 4.1%  │  ← Mono، الرقم 24px
│ من ٧ متاجر · فرق ٢٣٠│  ← رمادي 12px
│ [☐ أضف للمقارنة]    │
└─────────────────────┘
```
«فرق ٢٣٠» = الفرق بين أعلى وأدنى سعر. هاي أهم معلومة بمنصة مقارنة وأغلب المنافسين ما بيعرضوها.

**الترتيب:** الأقل سعرًا · الأكثر انخفاضًا · الأكثر مقارنة · الأحدث.

**حالة لا نتائج:** «ما في نتائج لـ "…". جرّب اسم أقصر، أو تصفّح [سامسونج] [آيفون] [شاومي].» — دعوة للفعل، مش اعتذار.

### 2.3 صفحة المنتج — Product Details

**الهدف:** يشوف المستخدم كل الأسعار، يفهم اتجاه السعر تاريخيًا، ويقرر: يشتري الآن (رابط تسويق) أو يحجز.

**التخطيط:** عمودان. يمين (60%) المحتوى، يسار (40%) بطاقة `sticky`.

**البطاقة اللاصقة:**
```
┌────────────────────────┐
│ أفضل سعر الآن          │
│ 1,150 ر.س              │  ← Mono 36px، أخضر
│ متجر التقنية · محدّث ٣ د│
│ ┌────────────────────┐ │
│ │  اذهب إلى المتجر   │ │  ← أخضر مصمت (Affiliate)
│ └────────────────────┘ │
│ ┌────────────────────┐ │
│ │   احجز مسبقًا      │ │  ← مفرغ بحد أخضر
│ └────────────────────┘ │
│  ♡ مفضلة   ⇄ قارن     │
└────────────────────────┘
```
زر «اذهب إلى المتجر» تحته سطر إفصاح صغير: «قد نحصل على عمولة من هذا الرابط. السعر لا يتغيّر عليك.»

**جدول المتاجر:** المتجر | السعر (Mono) | التوفر | آخر تحديث | إجراء.
الصف الأول (أرخص) بخلفية `--green-tint` وشارة «أقل سعر». المتاجر غير المتوفرة برمادي باهت مع حفظ مكانها.

**مخطط تاريخ السعر (Price_History):**
مخطط خطي، تبويبات ٣٠ يوم / ٩٠ يوم / سنة. الخط أخضر 2px مع تعبئة متدرّجة خفيفة. خط أفقي متقطع رمادي = المتوسط. علامة حمراء صغيرة عند أعلى سعر مسجّل. Tooltip: التاريخ + السعر + المتجر.
تحت المخطط سطر واحد بالنص: «أدنى سعر مسجّل: ١,٠٩٠ في ١٢ مارس.» — هاي الجملة أهم من المخطط نفسه.

**تنبيه السعر (Price Alert):**
صندوق داخل المحتوى: «نبّهني إذا نزل تحت [____] ر.س» + زر «فعّل التنبيه». يقترح تلقائيًا رقمًا = أدنى سعر تاريخي.

**المواصفات:** جدول من عمودين، صفوف مخططة بـ `--surface`، مجمّعة تحت عناوين (الشاشة، المعالج، الكاميرا، البطارية).

**التقييمات الموثّقة:**
رأس القسم: `4.6` بخط 40px + شريط + «٨٧ تقييمًا موثّقًا».
كل تقييم: اسم + ★ حمراء صغيرة + «تمّ الاستلام في ٤ يونيو». **ما في زر تقييم لأي زائر** — القسم يقبل فقط من عنده حجز مكتمل، وهاد بيتشرح بسطر واحد تحت العنوان.

### 2.4 المقارنة والتنقّل — Comparison & Navigation

**شريط المقارنة العائم:** يطلع من أسفل الشاشة لحظة اختيار أول منتج. ارتفاع 72px، خلفية `--ink`، نص أبيض. يعرض حتى ٤ صور مصغّرة + زر «قارن (٣)» أخضر + «مسح». يبقى ظاهرًا عبر كل الصفحات (Session).

**صفحة المقارنة:** جدول أفقي، **العمود الأول ثابت** (أسماء الخصائص) مع تمرير أفقي للباقي.
- في كل صف، القيمة الأفضل بخلفية `--green-tint` ووزن 600.
- مفتاح تبديل: «إخفاء المتشابه» — بيخفي الصفوف اللي كل القيم فيها متطابقة. هاي الميزة هي كل الفرق بين مقارنة مفيدة وجدول ممل.
- صف الأسعار في الأعلى وثابت عند التمرير العمودي.
- زر «احجز» أسفل كل عمود.

**الحد الأقصى ٤ منتجات.** لما يحاول يضيف الخامس: توست رمادي «المقارنة تتّسع لأربعة. أزل واحدًا أولًا.»

### 2.5 تدفّق الحجز — Booking Flow

هون بيظهر العنصر التوقيعي. مؤشر ثلاث نجمات أعلى الصفحة:

```
        ★ ─────── ☆ ─────── ☆
      اختيار      تأكيد     استلام
```
النجمة تُملأ بـ `--star` عند إكمال خطوتها.

**الخطوة ١ — اختيار:** المتجر (بطاقات فيها السعر + المسافة/العنوان)، اللون، السعة، الكمية (حد أقصى ٢). سطر تحذير هادئ بخلفية `--surface`: «الحجز يثبّت السعر والكمية لمدة ٤٨ ساعة. الإلغاء المتكرر يقيّد قدرتك على الحجز.» — نص أسود، بدون أيقونة إنذار. تحذير مش تهديد.

**الخطوة ٢ — التأكيد + قفل التزامن (Concurrency Lock):**
عند الضغط، الزر ما بيصير spinner عام. بيتحوّل لنص صريح: **«نثبّت لك القطعة…»** لمدة ما القفل شغّال.
- نجاح → انتقال فوري للخطوة ٣.
- فشل → البطاقة تهتز اهتزازة واحدة خفيفة، ورسالة حمراء: «نفدت الكمية أثناء التأكيد. ما تم خصم أي شيء.» + زر «اعرض متاجر أخرى بالسعر نفسه». الخطأ بيشرح شو صار وشو الخطوة الجاية — بدون اعتذار.

**الخطوة ٣ — الاستلام:**
رقم الحجز بخط Mono 28px + QR. تحته **العدّاد** — العنصر الأحمر الوحيد بالصفحة:
`٤٧:١٢:٣٣` بخط Mono، خلفية `--star-tint`، نص `--star`، ولابل صغير: «يُلغى الحجز تلقائيًا بعد».
تحته: عنوان المتجر + خريطة صغيرة + زر «إلغاء الحجز» نصّي رمادي (مش زر أحمر بارز — ما بدنا نشجّع الإلغاء).

**عند الإلغاء:** حوار تأكيد يعرض عدّاد الإلغاءات: «هذا إلغاؤك الثاني هذا الشهر. بعد الثالث يُعلَّق الحجز ١٤ يومًا.» شفافية مباشرة بدل عقوبة مفاجئة.

### 2.6 لوحة المستخدم — User Dashboard

تبويبات أفقية: **الملف · المفضلة · تنبيهات السعر · حجوزاتي · تقييماتي**

- **الملف:** صورة، اسم، بريد، جوال، لغة، تغيير كلمة المرور. عمود جانبي فيه «حالة الحساب: جيدة» — وإذا في تقييدات، بيصير «مقيّد حتى ٢٢ يوليو» بأحمر مع سبب واضح.
- **المفضلة:** شبكة بطاقات، كل بطاقة بتعرض السعر عند الإضافة مقابل السعر الآن، بسهم ملوّن. (قيمة حقيقية: بيشوف إذا انتظاره كان مربحًا.)
- **تنبيهات السعر:** جدول — المنتج | الحد المطلوب | السعر الحالي | الفارق | مفتاح تشغيل. الصفوف اللي وصلت الحد بخلفية `--green-tint` وشارة «تحقق ✓».
- **حجوزاتي:** بطاقات بحالات: `نشط` (أخضر) · `مكتمل` (رمادي) · `ملغي` (خط عبر النص). النشط بيعرض العدّاد.
- **الحالة الفارغة للمفضلة:** «لسّه ما تابعت أي جوال. تابع واحدًا ومنخبرك أول ما ينزل سعره.» + زر «تصفّح الأكثر انخفاضًا».

### 2.7 لوحة البائع — Vendor Dashboard

**نظرة عامة (٤ بطاقات KPI):** منتجات نشطة · حجوزات اليوم · معدّل التحويل · **نسبة الإلغاء** (الأخيرة تصبح حمراء فوق عتبة معيّنة).
تحتها مخطط عمودي: الحجوزات آخر ٣٠ يوم.

**منتجاتي:** جدول كثيف، تحرير مباشر (inline) لخانتين فقط: **السعر** و**المخزون**. الحفظ تلقائي مع تلميح صغير «حُفظ ١٤:٣٢». عمود «آخر تحديث» بيصير رمادي باهت لو مرّ أكتر من ٤٨ ساعة (ضغط لطيف لتحديث الأسعار).

**الاستيراد الجماعي (Bulk Import):** هاي الشاشة الأهم عند البائع، وأغلب المنصات بتفشل فيها.
```
١. ارفع الملف          [منطقة سحب] CSV / XLSX  · [نزّل القالب]
٢. طابق الأعمدة        عمودك «سعر البيع» ← حقلنا «price»
٣. المعاينة            أول ٥ صفوف كما ستُحفظ
٤. التقرير             ✓ ١٤٠ صفًا جاهزًا  ·  ✗ ٦ صفوف بها أخطاء
```
تقرير الأخطاء **بمستوى الصف**: «الصف ٢٣: السعر غير رقمي (`1,2o0`)». وزر «نزّل الصفوف الفاشلة فقط» ليصلحها ويعيد رفعها. الاستيراد الجزئي مسموح.

**الحجوزات:** قائمة + بحث بالرقم + زر «تم الاستلام» (بيغلق الحجز ويفتح للمستخدم حق التقييم).
**الربط التقني:** مفاتيح API، رابط Webhook، سجل آخر ١٠ محاولات مزامنة بحالاتها.

### 2.8 لوحة الإدارة — Admin

خلفية `--surface`، كثافة أعلى، حواف 8px، بدون رسوم زائدة. أداة عمل مش واجهة تسويق.

- **صحة النظام:** بطاقات لمهام الـ Cron والـ Scraping: اسم المهمة | آخر تشغيل | المدة | الحالة (✓ / ✗). المهام الفاشلة تُرفع لأعلى القائمة تلقائيًا.
- **المستخدمون والبائعون:** جداول مع إجراءات (توثيق، تعليق، إعادة تعيين). أزرار التعليق **نصّية حمراء**، مش أزرار مصمتة.
- **مراجعة التقييمات:** طابور المراجعة، كل تقييم بجانبه دليل الحجز المرتبط فيه.
- **الأمان:** سجل زمني — محاولات دخول فاشلة، تجاوزات Rate Limit، عناوين IP مشبوهة. فلترة بالفترة والنوع.
- **النسخ الاحتياطي:** «آخر نسخة ناجحة: اليوم ٠٣:٠٠ · ٤١٢ م.ب» + جدولة + زر استعادة خلف حوار تأكيد يطلب كتابة كلمة «استعادة».
- **SEO:** حالة الـ Sitemap، عدد الروابط، آخر توليد، زر إعادة توليد.

### 2.9 المصادقة — Auth

شاشة نصفين. النصف الأيسر: ثلاثة أشرطة لونية أفقية عريضة (أخضر / أبيض / أسود) بتحرك بطيء جدًا، وثلاث نجمات حمراء بالوسط. بدون نص تسويقي. النصف الأيمن: النموذج فقط.
حقول 48px، تسميات فوق الحقول (مش placeholder)، وزر أخضر عريض. رابط «تسجيل كبائع» منفصل ومميّز بحد أسفل.

**الأخطاء:** «البريد أو كلمة المرور غير صحيحة.» — جملة واحدة، تحت الحقل، حمراء. ما بنقول أي واحد فيهم الغلط (أمان).

### 2.10 حالات الفراغ والخطأ

| الحالة | النص |
|---|---|
| 404 | «هذه الصفحة غير موجودة. جرّب البحث عن المنتج الذي تريده.» + حقل بحث |
| خطأ خادم | «تعذّر تحميل الأسعار. نحاول مجددًا خلال ثوانٍ.» + مؤشر إعادة محاولة |
| بدون اتصال | «لا يوجد اتصال. الأسعار المعروضة قد تكون قديمة.» شريط علوي رمادي |
| نتائج صفر | دعوة لتصفّح فئات محددة، لا اعتذار |

القاعدة: الخطأ ما بيعتذر، وبيقول شو صار وشو الخطوة الجاية. الشاشة الفارغة دعوة للفعل.

## 3. مكتبة المكونات

| المكون | ملاحظات |
|---|---|
| زر أساسي | `--green` مصمت، ارتفاع 44px، حواف 8px، نص 500 |
| زر ثانوي | مفرغ بحد `--green` 1.5px |
| زر مدمّر | نصّي `--star` — أبدًا مصمت |
| خانة سعر | Mono 500، محاذاة يسار داخل الخلية |
| شارة انخفاض | `--green-tint` + سهم ↓ + النسبة |
| شارة ارتفاع | `--star-tint` + سهم ↑ |
| شارة «موثّق» | ★ حمراء 12px + نص 12px |
| عدّاد | Mono، `--star`، خلفية `--star-tint` |
| مؤشر النجوم | ثلاث نجمات، الحالية بحد نابض |
| هيكل تحميل | مستطيلات `--surface` بنبض، لا سبينر دوّار |
| شريط المقارنة | ثابت أسفل، `--ink` |

**الحد الأدنى غير القابل للتفاوض:** استجابة كاملة للجوال، تباين نصوص ≥ 4.5:1، تركيز لوحة المفاتيح ظاهر (حلقة خضراء 2px)، واحترام `prefers-reduced-motion` (التيكر بيوقف، النبض بيوقف).

## 4. برومبتات جاهزة لـ Claude Design

**الرسالة الأولى:** الصق القسم ١ كاملًا + هذا السطر:
> هذه هي الهوية البصرية. لا تنحرف عنها. سأعطيك صفحة واحدة في كل رسالة. اللغة عربية RTL بالكامل.

**بعدها، رسالة لكل صفحة:**
> ابنِ [الصفحة الرئيسية] حسب الوصف التالي حرفيًا: [الصق 2.1]. استخدم محتوى واقعيًا: أسماء جوالات حقيقية وأسعار بالريال السعودي. لا تستخدم Lorem ipsum. البطل هو شريط الأسعار الحيّ، وليس صورة منتج.

**للمراجعة بعد كل صفحة:**
> راجع الصفحة: هل الأحمر مستخدم خارج الحالات الأربع المسموحة؟ هل التقييمات معروضة بنجوم؟ هل في مكان ثاني استخدمت النجمة غير مؤشر الحجز وشارة التوثيق؟ صلّح إن وجدت.
